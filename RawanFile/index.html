Git Command Line Tutorial

Your essential guide to mastering Git from the terminal.

1. Introduction to Git

Git is a free and open-source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. It allows multiple developers to work on the same project simultaneously without overwriting each other's changes.

This tutorial will guide you through the fundamental Git commands you'll use daily.

2. Setting Up Git

Install Git

First, ensure Git is installed on your system. You can download it from the official Git website or use your system's package manager.

  • Windows: Download from git-scm.com.
  • macOS: Install with Homebrew (`brew install git`) or Xcode Command Line Tools.
  • Linux: Use your package manager (e.g., `sudo apt-get install git` for Debian/Ubuntu, `sudo yum install git` for Fedora/CentOS).

Configure Git

After installation, set your user name and email address. These will be attached to your commits.

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

3. Basic Git Commands

3.1. `git init` - Initialize a New Repository

To start tracking a new project with Git, navigate to your project directory and run:

git init

This creates a hidden `.git` directory, which is where Git stores all its tracking information.

3.2. `git add` - Stage Changes

After making changes to your files, you need to "stage" them before committing. Staging means telling Git which changes you want to include in your next commit.

To add a specific file:

git add filename.txt

To add all changes in the current directory:

git add .

3.3. `git commit` - Save Changes

A commit records a snapshot of your staged changes to the repository's history. Always include a clear and concise message describing your changes.

git commit -m "Your descriptive commit message"

3.4. `git status` - Check Repository Status

This command shows the current state of your working directory and staging area. It tells you which changes are staged, unstaged, or untracked.

git status

3.5. `git log` - View Commit History

To see a history of all commits in the current branch, including commit hash, author, date, and message:

git log

For a more condensed view:

git log --oneline

4. Working with Remote Repositories

Remote repositories are versions of your project hosted on the internet or network, like GitHub, GitLab, or Bitbucket. They allow collaboration and serve as a backup.

4.1. `git clone` - Download a Repository

To get a copy of an existing remote repository onto your local machine:

git clone <repository_url>

Example:

git clone https://github.com/user/repository.git

4.2. `git pull` - Fetch and Merge Changes

To download changes from the remote repository and automatically merge them into your current local branch:

git pull origin main

(`origin` is the default name for your remote, `main` is often the primary branch name)

4.3. `git push` - Upload Changes

To send your committed changes from your local repository to the remote repository:

git push origin main

The first time you push a new local branch, you might need to set the upstream:

git push -u origin main

5. Branching and Merging

Branching allows you to diverge from the main line of development and work on new features or bug fixes without affecting the stable codebase.

5.1. `git branch` - Manage Branches

To list all local branches:

git branch

To create a new branch:

git branch new-feature

5.2. `git checkout` - Switch Branches

To switch to an existing branch:

git checkout new-feature

To create a new branch and switch to it immediately (common shortcut):

git checkout -b another-new-feature

5.3. `git merge` - Integrate Changes

To integrate changes from one branch into your current branch (e.g., merge `new-feature` into `main`):

git checkout main
git merge new-feature

After merging, you can delete the feature branch if it's no longer needed:

git branch -d new-feature

6. Conclusion

This tutorial covered the most frequently used Git commands. Git has a much deeper set of features, but mastering these basics will enable you to work effectively on most projects.

Key Takeaways:

  • Initialize: `git init`
  • Track: `git add .`
  • Save: `git commit -m "message"`
  • Check Status: `git status`
  • View History: `git log`
  • Collaborate: `git clone`, `git pull`, `git push`
  • Organize: `git branch`, `git checkout`, `git merge`

Keep practicing, and don't hesitate to consult the official Git documentation or online resources when you encounter new scenarios!
